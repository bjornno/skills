# Specs

- **`OVERVIEW.md`** — what the product/service does and main capabilities (plain language); short developer orientation. Read this first.
- **Feature / area** — `specs/<feature-slug>/`: start with **`SPEC.md`** only; optionally split into **`intent.md`** (stable), **`experience.md`** (flows, stories, acceptance), **`constraints.md`** (technical requirements), **`ui.md`** (hints for UI/app devs when this repo is backend-only). **`SPEC.md`** stays the entry and holds **Changelog**.

Updated alongside meaningful code changes. Workflow: **Storyline** — `.cursor/skills/storyline/SKILL.md`. Optional chat shortcuts: type **`/`** and pick **`storyline-*`** from `.cursor/commands/`.
